DOCman API

Documents
Documents CollectionDocumentRetrieve a DocumentCreate or Edit a DocumentRemove a Document

http://demo.joomlatools.com/joomla3/

The DOCman Web API is a Restful JSON API. The default JSON response documents are inspired by the JSON API specification. For a deeper look at how the core JSON format actually gets built have a look our write up about JSON at http://guides.nooku.org/JSON.html.

The responses used below are taken directly from http://demo.joomlatools.com/joomla3/ and can be duplicated there.

A Valid Item ID

DOCman requests must be associated with a valid Joomla! menu item (Itemid), and the following API documentation assumes you will add it with each request.**

PHP and Events API

Included in each request type is the PHP API analogue. It shows you how you would do the same operation from inside the code. Also, a list of relevant events published through the Event API is provided. We cover these in some detail in the DOCman plugins and Framework Plugins sections.

Documents 

Documents Collection 

GET
/?option=com_docman&{&view,slug,created_by,category,limit,offset,sort,direction}
List a subset of Documents
PHP API

    // Get a list of documents through the controller for a given set of parameters.
    $documents = KObjectManager::getInstance()
                        ->getObject('com://docman.controller.document')
                        ->category(24)
                        ->sort('title')
                        ->browse();

Event API

BeforeAfter
onBeforeDocmanControllerDocumentRenderonAfterDocmanControllerDocumentRender
onBeforeDocmanControllerDocumentBrowseonAfterDocmanControllerDocumentBrowse
onBeforeDocmanModelDocumentFetchonAfterDocmanModelDocumentFetch
onBeforeDocmanModelDocumentCountonAfterDocmanModelDocumentCount

See: KControllerModel::_actionBrowse

  • Parameters
  • view
    string (required) Example: documents

    The view choices are filteredlist, userlist, list or documents.

    slug
    string (optional) Example: baking

    The slug is used to specify the category that the documents should belongs to.

    category
    int (optional) Example: 24

    Is used to specify the category that the list of documents should belong to.

    sort
    string (optional) Example: title

    Column used to sort the results in the response.

    direction
    string (optional) Example: asc

    The direction of the resulting sort. The default is 'asc’.

    offset
    int (optional) Example: 6

    Defines the beginning of the subset of entities.

    limit
    int (optional) Example: 2

    Defines the number entities that should be returned in the response.

POST
/?option=com_docman&view=document{&id}
Create or Edit a Document
PHP API

    // Create a new document
    $data = array("title" => "Cake",
            "docman_category_id" => 24,
            "image": "recipes/cake-110934_640.jpg"
            ); // and more
    $controller = KObjectManager::getInstance()
                            ->getObject('com://docman.controller.document');
    // add a document
    $newDocument = $controller->add($data);
    // edit an existing document
    $updatedDocument = $controller->id(74)->edit($data);

/?option=com_docman&view=document{&id}

PHP API

    // Loads the document row and then deletes that row object.
    $documents = KObjectManager::getInstance()
                        ->getObject('com://docman.controller.document')
                        ->id(74)
                        ->delete();
